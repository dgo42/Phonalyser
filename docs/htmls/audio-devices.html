Browser audio device probe

Enumerates input / output audio devices and probes which sample rates the browser will actually grant. Bit depth is not exposed by the Web platform (Web Audio is 32-bit float after the OS mixer), and there is no exclusive mode — so this shows the ceiling a pure browser tab can reach, not what the hardware can do.

Click Scan and allow microphone access — without permission the OS hides device labels. The scan opens each input device once per candidate rate, so it takes a few seconds.

2. Input sample-rate probe (getUserMedia + track.getSettings())

For each input device we request sampleRate: {exact: r} and report the rate the browser actually granted. ✓ = granted at the requested rate, ≈ = opened but resampled to a different rate, ✗ = refused.

4. What the platform does NOT expose

  • Bit depth — none. Capture is float32 post-mixer; no 16/24-bit raw PCM.
  • Exclusive mode — none. Every stream goes through the shared OS mixer + its resampler.
  • Per-output-device formats — not queryable; setSinkId only routes.
